# Car-ing is Sharing: LLM Prototype Pipeline

This repository contains a prototype NLP pipeline developed for **"Car-ing is sharing"**, a car sales and rental company. The goal of this project is to leverage pre-trained Large Language Models (LLMs) to automate and optimize customer inquiry handling, sentiment tracking, translations, and information extraction.

---

## 🚀 Project Overview

The prototype executes four core tasks proposed by the CTO to handle diverse customer-related data:
1. **Sentiment Analysis & Evaluation:** Classifying customer reviews and evaluating model performance against ground truth labels.
2. **Translation & Quality Assessment:** Translating English reviews to Spanish and evaluating accuracy using BLEU scores.
3. **Extractive Question Answering:** Pulling specific, brand-focused insights directly from customer text.
4. **Text Summarization:** Condensing lengthy reviews into concise, actionable summaries.

---

## 🛠️ Tasks & Technical Implementation

### Task 1: Sentiment Classification & Evaluation
* **Objective:** Classify the sentiment of five car reviews in `car_reviews.csv` and evaluate performance.
* **Outputs:** 
  * `predicted_labels`: Raw model classification outputs.
  * `predictions`: Mapped binary integer labels (`0` for negative, `1` for positive).
  * `accuracy_result`: The calculated accuracy score of the predictions.
  * `f1_result`: The calculated F1 score of the predictions.

### Task 2: English-to-Spanish Translation
* **Objective:** Translate the first two sentences of the first review to support expanding Spanish-speaking market demographics.
* **Evaluation:** Assess translation quality via BLEU score using `reference_translations.txt`.
* **Outputs:**
  * `translated_review`: The Spanish translation generated by the LLM.
  * `bleu_score`: The resulting BLEU score metric.

### Task 3: Extractive Question Answering
* **Objective:** Extract specific brand feedback from the 2nd review using a specialized QA model (`deepset/minilm-uncased-squad2`).
* **Inputs:** `question` ("What did he like about the brand?") and `context` (the 2nd review).
* **Outputs:** 
  * `answer`: The exact text snippet extracted by the LLM.

### Task 4: Review Summarization
* **Objective:** Condense the final review in the dataset to capture key takeaways efficiently.
* **Constraint:** Length targeted to approximately 50–55 tokens.
* **Outputs:**
  * `summarized_text`: The finalized short-form summary.
